## 1. 신규 프로젝트 등록 체크리스트

### 1.1 정보 수집 단계

#### 기본 정보
- [ ] 프로젝트 정식 명칭 확인
- [ ] 고객사/기관명 확인
- [ ] 프로젝트 유형 확인 (홈페이지/플랫폼/시스템/웹앱)
- [ ] 카테고리 분류 (4대 카테고리 중 하나)
- [ ] 실제 서비스/시범/MVP 상태 확인

#### 기술 정보
- [ ] 사용 기술 스택 리스트업
- [ ] 데모 URL 확인 (있는 경우)
- [ ] 핵심 기능 3가지 정리
- [ ] 적합 고객/타겟 정의

#### 권한 및 공개 수준
- [ ] 포트폴리오 공개 가능 여부 확인
- [ ] 데모 URL 공개 가능 여부 확인
- [ ] 스크린샷 사용 승인
- [ ] 고객사 로고 사용 승인 (필요 시)

### 1.2 식별자 생성 단계

#### ID 생성
```
체크 항목:
- [ ] Prefix 선택 (mvp/gen/jdx/grok/ketri/baikal/mxten/new/pub/music/figma/bank/fuel)
- [ ] 번호 부여 (해당 prefix 내 중복 확인)
- [ ] 형식 검증: {prefix}-{number} (예: ketri-01)
- [ ] 전체 프로젝트 목록에서 ID 중복 확인
```

**ID 생성 예시:**
```typescript
// ✅ 올바른 예
id: 'ketri-01'    // 연구소 프로젝트
id: 'gen-30'      // 범용 프로젝트
id: 'jdx-01'      // AI/교육 프로젝트

// ❌ 잘못된 예
id: 'Ketri-01'    // 대문자 사용
id: 'ketri_01'    // 밑줄 사용
id: 'ketri'       // 번호 누락
```

#### Slug 생성
```
체크 항목:
- [ ] 영문 소문자 사용
- [ ] 하이픈(-) 으로 단어 구분
- [ ] 의미 있는 키워드 포함
- [ ] 50자 이내
- [ ] 전체 프로젝트에서 slug 중복 확인
```

**Slug 생성 예시:**
```typescript
// ✅ 올바른 예
slug: 'ketri-homepage'           // 기관명-타입
slug: 'vibe-office-hub'          // 서비스명 그대로
slug: 'library-reservation'      // 기능-타입

// ❌ 잘못된 예
slug: 'ketri'                    // 너무 짧음
slug: 'Ketri-Homepage'           // 대문자 사용
slug: 'ketri_homepage'           // 밑줄 사용
slug: '한국환경안전연구소'       // 한글 사용
```

#### Title 작성
```
체크 항목:
- [ ] 정식 명칭 사용
- [ ] 50자 이내
- [ ] 불필요한 수식어 제거
- [ ] 일관된 형식 유지
```

### 1.3 데이터 입력 단계

#### portfolio.ts 업데이트

```typescript
{
  // 식별자
  id: 'ketri-01',                               // ✅ 확인
  title: '한국환경안전연구소 공식 홈페이지',    // ✅ 확인
  slug: 'ketri-homepage',                        // ✅ 확인
  
  // 분류
  category: '기업·기관 홈페이지',               // ✅ 4대 카테고리 중 하나
  
  // 설명
  oneLiner: '공공 기관의 신뢰성을 강화하는 공식 웹사이트',  // ✅ 50자 이내
  features: [                                    // ✅ 정확히 3개
    '연구 성과 및 자료실',
    '공지사항·뉴스 관리',
    '기관 소개 및 조직도'
  ],
  fitFor: '연구소·협회, 공공기관, 비영리단체',  // ✅ 타겟 명확
  
  // URL 및 이미지
  demoUrl: 'https://ketri-project-01.vercel.app/',  // ✅ 선택 사항
  thumbnail: '/images/thumbnails/ketri-homepage.webp',  // ✅ slug 일치
  screenshots: [                                 // ✅ 최소 1개
    '/images/screenshots/ketri-homepage-01.webp'
  ],
  
  // 기술 및 태그
  stack: ['React', 'TypeScript', 'TailwindCSS', 'Firebase'],  // ✅ 주요 기술
  tags: ['연구소', '공공기관', '환경', '안전'],  // ✅ 검색 키워드
  
  // 메타 정보
  featured: false,                               // ✅ Featured 여부
  yearMonth: '2026-01'                           // ✅ YYYY-MM 형식
}
```

**체크 항목:**
- [ ] 모든 필수 필드 입력 완료
- [ ] thumbnail 경로가 slug와 일치
- [ ] features 배열이 정확히 3개
- [ ] category가 4대 카테고리 중 하나
- [ ] yearMonth 형식이 YYYY-MM

- [ ] 형식: WebP
- [ ] 용량: 300KB 이하
- [ ] 저장 위치: /public/images/screenshots/

## 3. Featured 프로젝트 선정 체크리스트

### 3.1 기본 조건 (필수 - 모두 충족)

```
- [ ] 데모 URL이 정상 작동하는가?
- [ ] 고품질 썸네일이 있는가? (800×600, WebP)
- [ ] 프로젝트 설명이 완비되었는가? (oneLiner, features, fitFor)
- [ ] 외부 공개 가능한가? (전체 공개 또는 제한 공개)
```

---

### 3.2 우대 조건 (2개 이상 충족 권장)

```
- [ ] 실제 서비스로 운영 중인가?
- [ ] 고객사 인지도가 높은가?
- [ ] 기술 스택이 다양한가? (3개 이상)
- [ ] 해당 카테고리를 대표할 수 있는가?
- [ ] 최근 6개월 이내 프로젝트인가?
- [ ] 복잡도가 높은가? (난이도 medium 이상)
- [ ] 비즈니스 가치가 명확한가?
```

---

### 3.3 카테고리 균형 확인

```
현재 Featured 12개 배분:
- [ ] 기업·기관 홈페이지: 5개 (42%)
- [ ] 교육·AI 플랫폼: 2개 (17%)
- [ ] SaaS·업무자동화: 2개 (17%)
- [ ] 공공·예약·문화: 3개 (25%)

신규 Featured 추가 시:
- [ ] 카테고리 배분이 적절한가?
- [ ] 기존 프로젝트와 유사도가 낮은가?
- [ ] 포트폴리오 다양성에 기여하는가?
```

---

### 3.4 경쟁 프로젝트 비교

```
비교 기준:
- [ ] 기술적 완성도
- [ ] 디자인 품질
- [ ] 비즈니스 영향력
- [ ] 고객사 만족도
- [ ] 유지보수 현황

교체 결정:
- [ ] 신규 프로젝트가 기존 Featured보다 우수한가?
- [ ] 교체 시기가 적절한가?
- [ ] 교체 후 포트폴리오 균형이 유지되는가?
```
